## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI→Cache (optimistic update) | Temp entries inserted into TanStack Query cache — never sent to server with temp ID |
| Cache→Server (onSettled) | Query invalidation triggers refetch — server truth always wins |
| User→Dashboard (undo) | Undo triggers delete mutation — userId validated server-side via Supabase session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04 | Tampering | Optimistic temp entry in cache | accept | Temp entries use `temp-` prefix; server validates real entries; onSettled refetch replaces temp with server truth |
| T-11-05 | Information Disclosure | Content preview in EntryCard | accept | Content is user's own data; preview truncation is UX, not security; full content only visible to authenticated user |
| T-11-06 | Denial of Service | Large entry list scroll performance | accept | Typical usage: 10 entries/day max (rate limit); AnimatePresence handles list rendering; no virtualization needed at this scale |
| T-11-SC | Tampering | npm installs | mitigate | No new packages installed in this plan |
</threat_model>
